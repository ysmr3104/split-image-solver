## 2. 失敗の根本原因分析

### 原因1: 正距投影（gnomonic/TAN）の非線形性

14mm（FOV ~147°）では、画像端のピクセルスケールが中心と比較して**2倍以上**異なる:

```
中心:        54.4 arcsec/pixel
コーナー:   ~113 arcsec/pixel（2.1倍）
エッジ端:   ~87-95 arcsec/pixel（1.6-1.7倍）
```

タイルごとの補正済みスケールヒントを実装済みだが、solve-field のスケール範囲 ±20-40% では依然として不足するケースがある。

### 原因2: レンズ歪曲収差

14mm超広角レクティリニアレンズは顕著な**樽型歪曲収差**を持つ:
- 星のパターン（quad）がカタログと幾何学的に不一致
- タイル分割により歪みが不均一に分配される（中心タイルは歪み小、端タイルは歪み大）
- solve-field の SIP 多項式（デフォルト次数2）ではタイル内の局所歪みを十分にモデル化できない

### 原因3: インデックスファイルのクワッドスケール制限

| インデックスシリーズ | クワッドスケール範囲 | 対象FOV |
|-------|-------|-------|
| 4110 | 2-2.8 arcmin | ~10' |
| ... | ... | ... |
| 4119 | 22-35 arcmin | ~2-3° |
| 4214-4219 | Tycho2系 | 要確認 |

エッジタイルの実効FOVが30-40°の場合、タイル内に存在するクワッド（4星の組み合わせ）が35'より大きくなり、どのインデックスにもマッチしない。

### 原因4: 地上景

下端45%は地上景であり、星がほとんど写っていないためソルブ不可能。これは改善の余地がない。

## 3. カメラ・レンズ詳細情報の活用可能性

ユーザーが提供可能な詳細情報と、各パラメータのプレートソルブへの影響度を分析する。

### 3.1 各パラメータの影響度

| パラメータ | 影響度 | 用途 | 説明 |
|-----------|--------|------|------|
| **ピクセルピッチ** (μm) | **高** | スケール計算 | `arcsec/px = 206.265 * pitch / focal_length`。正確なスケールヒントに不可欠 |
| **焦点距離** (mm) | **高** | スケール計算、FOV計算 | ピクセルスケールとFOVの基本パラメータ |
| **センサーサイズ** (mm) | **中** | FOV計算、ピクセルピッチ推定 | `pixel_pitch = sensor_width / pixel_count` のフォールバック計算に使用 |
| **レンズ有効口径** (mm) | **低** | 直接影響なし | レンズ名の特定に間接的に役立つ |
| **撮影時の絞り** (f/) | **低〜中** | lensfun歪み補正 | lensfunでビネット/歪み補正を行う場合に必要。ソルブの幾何学には直接影響しない |

### 3.2 ピクセルピッチと焦点距離

**最重要パラメータ**。正確なピクセルスケール計算の根幹。

```python
# 現在の計算方法（main.py）
pixel_scale = 206.265 * (sensor_width / image_width) / focal_length * 1000

# ピクセルピッチ直接指定（より正確）
pixel_scale = 206.265 * pixel_pitch_um / focal_length
```

**注意**: drizzle後の画像は元の物理ピクセルピッチとは異なるため、`--pixel-scale` での直接指定が最も確実。

#### α7RV の場合
- ネイティブ解像度: 9504x6336 (61MP)
- ピクセルピッチ: 3.76 μm
- drizzle後 9728x6656: 実効ピクセルピッチ ≈ 3.67 μm（9504/9728 * 3.76）
- 14mm での理論スケール: `206.265 * 3.67 / 14 = 54.0 arcsec/pixel`（現在使用中の54.4と近い）

### 3.3 有効口径・絞り

プレートソルブの幾何学的精度には**直接影響しない**。ただし:

- **星像 SNR**: 開放ではより暗い星が写り、星の検出数が増える（ソルブ成功率に間接的影響）
- **ビネット（周辺減光）**: 開放では周辺減光が強くなり、エッジタイルの星検出が減少する可能性
- **lensfun歪み補正**: `mod.initialize(focal_length, aperture, ...)` のパラメータとして必要

### 3.4 レンズモデル名（最重要の追加情報）

具体的なレンズモデル名が最も有用。理由:
1. **lensfunデータベース照合**: レンズの歪み係数を自動取得可能
2. **歪み特性の把握**: 樽型か糸巻き型か、歪み量の程度を事前に把握
3. **口径蝕パターン**: エッジでの星像品質の予測

## 4. レンズ歪み補正による改善提案

### 4.1 方法A: lensfun/lensfunpy による事前歪み補正（最も有望）

#### 概要

画像分割前にレンズの歪曲収差を除去し、正距投影（TAN）に近い画像を生成してからタイル分割・ソルブする。

#### 対応状況

**lensfunデータベースに登録済みの14mm レンズ**:
- Samyang 14mm f/2.8 AE ED AS IF UMC
- Sigma 14mm f/1.8 DG HSM | Art
- Sigma 14mm f/2.8 EX / EX Aspherical HSM

**Sony FE 14mm F1.8 GM は未登録の可能性あり**。その場合はユーザーが手動でキャリブレーションデータを作成する必要がある。

#### 実装案

```python
# main.py の Step 1.5 として挿入
import lensfunpy
import cv2

def undistort_image(image_data, cam_maker, cam_model,
                    lens_maker, lens_model,
                    focal_length, aperture):
    db = lensfunpy.Database()
    cam = db.find_cameras(cam_maker, cam_model)[0]
    lens = db.find_lenses(cam, lens_maker, lens_model)[0]

    height, width = image_data.shape[:2]
    mod = lensfunpy.Modifier(lens, cam.crop_factor, width, height)
    mod.initialize(focal_length, aperture, float('inf'))

    undist_coords = mod.apply_geometry_distortion()

    # チャネルごとにリマップ
    result = np.empty_like(image_data)
    for c in range(image_data.shape[2]):
        result[:, :, c] = cv2.remap(
            image_data[:, :, c], undist_coords, None,
            cv2.INTER_LANCZOS4)
    return result
```

#### 期待効果
- エッジ/コーナータイルでの星パターン歪みが軽減
- quad マッチング成功率の向上
- ソルブ成功タイル数: 11 → 推定 15-25 タイル

#### 制約・課題
- lensfunのレンズプロファイル精度はコミュニティ依存
- drizzle後の画像に適用する場合、ピクセルスケールの調整が必要
- 15mm魚眼レンズ（等距離射影）の場合、lensfunの歪みモデルが異なる可能性

### 4.2 方法B: solve-field の --predistort オプション

#### 概要

成功タイルの SIP 歪み係数を使い、失敗タイルのソルブ時に歪みを事前除去する。

```bash
solve-field --predistort center_tile.wcs \
  --crpix-center --tweak-order 4 \
  edge_tile.fits
```

#### 利点
- lensfun不要（外部依存なし）
- 画像自体の歪みデータ（SIP）を直接利用
- 実装が比較的簡単

#### 課題
- 1stパスで成功するタイルが必要（鶏と卵問題）
- タイル間で歪みの向きが異なるため、中心タイルのSIPがエッジタイルに適切とは限らない
- **タイル単位のSIPは画像全体の歪みモデルではない**

### 4.3 方法C: solve-field オプション最適化（即座に試行可能）

#### 現在のコードに追加すべきオプション

```python
# astrometry_local_solver.py の cmd 構築部分に追加
cmd.extend([
    '--crpix-center',       # 歪みの中心点を画像中心に固定
    '--tweak-order', '4',   # SIP次数を2→4に引き上げ
])
```

#### 期待効果
- `--crpix-center`: レンズの光学中心を基準にした歪みモデリング
- `--tweak-order 4`: より高次の歪みをフィッティング（精度 ~8" → ~2"）
- 既存のソルブ成功タイルでも精度が向上し、WCS統合の品質が改善

### 4.4 方法D: 大スケール用インデックスファイルの追加

#### 現状のインデックスファイル
- 4110-4119: 2000/Tycho2系、クワッドスケール 2'-35'
- 4214-4219: Tycho2系

#### 不足しているインデックス
エッジタイル（FOV 30-40°）のクワッドスケールをカバーするためのインデックス:
- **4100-4107**: プリセット0-7（大スケール、60'以上のクワッド）
- カスタムインデックス: `-d 3`（3星特徴量）で構築

```bash
# 大スケール用インデックスのダウンロード例
# http://data.astrometry.net/4100/ から
wget http://data.astrometry.net/4100/index-4107.fits  # ~30-60 arcmin quads
```

## 5. 15mm魚眼レンズ対応の考慮

### 5.1 魚眼レンズの投影モデル

魚眼レンズは正距投影（gnomonic/TAN）とは根本的に異なる投影を使用する:

| 投影タイプ | 数式 | 特徴 | 代表レンズ |
|-----------|------|------|-----------|
| **等距離射影** (equidistant) | r = f * θ | θに比例、全天可能 | 多くの魚眼 |
| **等立体角射影** (equisolid) | r = 2f * sin(θ/2) | 面積保存 | Sigma 8mm等 |
| **正射影** (orthographic) | r = f * sin(θ) | 周辺圧縮 | 稀 |
| **立体射影** (stereographic) | r = 2f * tan(θ/2) | 形状保存 | Samyang 12mm等 |
| **正距投影** (gnomonic/TAN) | r = f * tan(θ) | 直線保存、90°限界 | 通常のレンズ |

**15mm魚眼の場合**: FOV ~180° の場合は等距離射影が最も一般的。FOV ~110-120° の対角線魚眼なら等距離射影または等立体角射影。

### 5.2 魚眼レンズがプレートソルブに与える影響

1. **ピクセルスケールの非線形性がTANよりさらに極端**:
   - gnomonic: `scale ∝ 1/cos²(θ)` → θ→90°で無限大
   - equidistant: `scale ∝ θ/sin(θ)` → 中心と端でスケール差が小さい

2. **星のパターン変形**: 魚眼投影のクワッドパターンはTANカタログと大きく異なる

3. **solve-field のWCSモデルはTAN投影を前提**: SIP多項式で歪みを補正するが、魚眼の大きな歪みにはSIP次数5以上が必要で、不安定になりがち

### 5.3 魚眼レンズへの対応戦略

#### 戦略A: 歪み補正してTANに変換してからソルブ

```
魚眼画像 → lensfun/手動で等距投影に変換 → 正距投影画像 → タイル分割 → ソルブ
```

- **利点**: 既存のパイプラインがそのまま使える
- **課題**: FOV > 90° の場合、正距投影への変換で画像端が極端に引き伸ばされる

#### 戦略B: 立体射影（stereographic/STG）や正距方位図法（zenithal equidistant/ZEA）への変換

```
魚眼画像 → 立体射影に変換 → タイル分割 → ソルブ（CTYPE=RA---STG）
```

- **利点**: FOV > 90° でも安定した変換
- **課題**: solve-field は TAN+SIP を前提としているため、STG/ZEA での直接ソルブには追加対応が必要

#### 戦略C: 小タイル分割+高密度グリッド

```
魚眼画像 → 12x12 以上のグリッド分割 → 各タイルのFOVを <15° に抑制 → ソルブ
```

- **利点**: 各タイルが十分に小さければ、どの投影でもTAN近似が成り立つ
- **課題**: タイル数増加による処理時間増大、エッジタイルの星数減少

#### 推奨アプローチ

**lensfunでの事前歪み補正（戦略A）+ 適切なグリッドサイズ（戦略C）の組み合わせ** が最も実用的:

1. lensfun で魚眼歪みを正距投影に補正
2. 補正後の画像を適切なグリッド（10x10以上）で分割
3. 各タイルのFOVを ~15° 以下に抑制
4. 通常のTAN+SIPでソルブ
